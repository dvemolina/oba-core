Booking → Session Flow Structure

How should the booking form and session setup relate to each other?

A

Inline progressive form Recommended

Booking fields at top, then a "Sessions" section unfolds below — visually nested/indented to show they belong to this booking. Session details (date, time, instructor, level, participants count) all on same page. Submit once.

Pros

  • One submit, no back-and-forth
  • Visual nesting shows hierarchy
  • Works well on mobile
  • Sessions feel "inside" the booking

Cons

  • Long form for complex bookings
B

Two-step wizard with persistent booking header

Step 1 = booking contract (service, client, participants count, notes). Step 2 = session(s). A persistent "booking summary bar" stays visible at top of step 2 so the hierarchy is always clear.

Pros

  • Cleaner, shorter screens
  • Header reinforces booking → session hierarchy

Cons

  • Still 2 steps / 2 submits
  • More complex to build
C

Current flow, better labeling only

Keep the separate booking → session steps but add clear visual language: a "Booking #123 envelope" wraps the sessions list, and each session card shows it belongs to the booking. No form changes.

Pros

  • Minimal dev work

Cons

  • Doesn't solve the back-and-forth problem
  • Owner still has to do extra steps
