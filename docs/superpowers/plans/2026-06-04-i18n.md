# i18n — Spanish/English Translations

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract all hardcoded English UI strings into Paraglide message keys, add Spanish translations (base locale), English translations, and add a language toggle in the nav.

**Architecture:** Paraglide already configured with `baseLocale: "es"`, cookie strategy (`PARAGLIDE_LOCALE`), messages in `/messages/{locale}.json`. No URL rewriting needed — cookie switch + page reload changes locale. Language toggle sets cookie via a server action, then redirects back.

**Tech Stack:** SvelteKit 2, Svelte 5 runes, Paraglide v2, Tailwind CSS v4

**Import pattern in every svelte file:**
```svelte
<script>
  import * as m from '$lib/paraglide/messages';
</script>
<!-- usage: {m.key_name()} or {m.key_with_param({ param: value })} -->
```

Use a simple GET endpoint that sets the cookie and redirects back:

```typescript
import { redirect } from '@sveltejs/kit';
import { availableLanguageTags } from '$lib/paraglide/runtime';
import type { RequestHandler } from './$types';

export const GET: RequestHandler = async ({ url, cookies }) => {
	const locale = url.searchParams.get('locale') ?? 'es';
	const referer = url.searchParams.get('from') ?? '/';

	if (availableLanguageTags.includes(locale as any)) {
		cookies.set('PARAGLIDE_LOCALE', locale, {
			path: '/',
			maxAge: 34560000,
			sameSite: 'lax'
		});
	}

	redirect(302, referer);
};
```
